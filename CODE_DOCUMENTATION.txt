1. PROJECT OVERVIEW
================================================================================

An AI-powered autonomous drone surveillance system that monitors industrial
properties 24/7. It processes drone camera frames through a multi-stage AI
pipeline: Vision Language Model analysis, LangChain ReAct agent reasoning,
rule-based alert evaluation, and dual-database indexing for structured and
semantic search.

Key Technologies:
  - Python 3.11
  - OpenAI GPT-4o-mini (Vision + Text)
  - LangChain + LangGraph (ReAct Agent)
  - YOLOv8 + DeepSORT (Object Detection + Tracking)
  - SQLite (Structured Storage)
  - ChromaDB (Semantic Vector Search)
  - FastAPI + SSE (Web Backend)

4. SIMULATORS
================================================================================

4.1 frames.py — Frame Simulator
--------------------------------
Function: get_simulated_frames() -> list[dict]
  Returns 30 drone surveillance frames covering a full 24-hour patrol cycle.
  Each frame is a dictionary with:
    - frame_id (int): Sequential ID from 1 to 30
    - time (str): Timestamp in HH:MM format (e.g., "06:00")
    - location (str): Patrol location name
    - description (str): Rich text description of what the camera sees

  Frame Categories:
    Frames 1-5   (06:00-07:00)  Early morning, employee arrivals
    Frames 6-10  (08:30-10:30)  Business hours, deliveries
    Frames 11-14 (11:30-13:00)  Midday, routine + suspicious vehicle
    Frames 15-20 (14:30-17:30)  Afternoon, suspicious persons, departures
    Frames 21-24 (18:30-21:30)  Evening, reduced activity
    Frames 25-30 (22:15-01:00)  Night, breach attempts, theft

Function: get_frame_by_id(frame_id: int) -> dict | None
  Returns a specific frame by its ID.

Function: get_frames_by_time_range(start: str, end: str) -> list[dict]
  Returns frames within a time range. Handles overnight ranges.

4.2 telemetry.py — Telemetry Simulator
---------------------------------------
Class: TelemetrySimulator
  Simulates drone telemetry data for each patrol frame.

  __init__(drone_id: str)
    Initializes with 100% battery and mission_start flag.

  get_telemetry(time_str: str, location: str) -> dict
    Returns telemetry dictionary:
      - drone_id: Drone identifier
      - time: Timestamp
      - location: Patrol location
      - altitude_m: 8-25m (varies by location type)
      - battery_pct: Drains 1-3% per frame (starts 95-100%)
      - wind_speed_kmh: 1-15 km/h (calmer at night)
      - gps_signal: "strong" (90%) or "moderate" (10%)
      - status: "patrolling"

  Altitude Logic:
    Gate areas:      10-15m
    Perimeter fence: 15-25m
    Parking lots:    12-18m
    Warehouse/Dock:  8-14m
    Server room:     10-15m

5. ANALYSIS — vlm_analyzer.py
================================================================================

Purpose: Sends frame descriptions to GPT-4o-mini for structured analysis.

Function: analyze_frame(description: str, telemetry: dict) -> dict
  Input:
    - description: Text description of the frame (from simulator or Vision API)
    - telemetry: Drone telemetry data

  Process:
    1. Builds a security analyst system prompt
    2. Constructs a user message with frame description + telemetry
    3. Calls GPT-4o-mini API requesting JSON output
    4. Parses the structured response

  Output (dict):
    - objects (list[str]): Detected objects (e.g., ["Person", "Blue Ford F150"])
    - event_type (str): Classification (e.g., "suspicious_activity", "delivery")
    - risk_level (str): "low", "medium", "high", or "critical"
    - is_suspicious (bool): Whether the frame contains suspicious activity
    - description (str): AI-generated analysis summary
    - recommended_action (str): Suggested security response

  Fallback:
    If the API call fails, a keyword-based fallback parser extracts basic
    analysis from the description text using pattern matching.

6. AGENT — security_agent.py, tools.py, prompts.py
================================================================================

6.1 security_agent.py — LangGraph ReAct Agent
----------------------------------------------
Class: SecurityAgent
  The core AI agent that processes frames, makes security decisions, and
  maintains awareness of previous events.

  __init__(sqlite_store, chroma_store, alert_manager)
    1. Initializes tool dependencies via init_tools()
    2. Creates ChatOpenAI LLM (gpt-4o-mini, temperature=0.1)
    3. Builds LangGraph ReAct agent with 4 tools
    4. Sets up conversation memory (last 20 messages)

  process_frame(frame: dict, analysis: dict, telemetry: dict) -> dict
    1. Constructs a detailed prompt with frame data + analysis + telemetry
    2. Appends to conversation memory
    3. Invokes the LangGraph agent (which may call tools autonomously)
    4. Extracts the agent's response
    5. Returns dict with agent_response and any tool results

  get_summary() -> str
    Invokes the agent with a summary prompt to generate a session report.

6.2 tools.py — 4 Custom LangChain Tools
-----------------------------------------
All tools are decorated with @tool and accept JSON string inputs.

Tool 1: log_event(event_json: str) -> str
  Logs a security event to both databases.
  - Writes to SQLite (frames table)
  - Writes to ChromaDB (vector embeddings)
  - Tracks vehicle re-entries (vehicle_log table)
  - Returns confirmation with re-entry warnings

Tool 2: check_alert_rules(analysis_json: str) -> str
  Evaluates all 8 alert rules against a frame analysis.
  - Gets vehicle context from SQLite
  - Calls AlertManager.evaluate_rules()
  - Logs triggered alerts to SQLite
  - Returns list of triggered alerts or "No alerts triggered"

Tool 3: query_past_events(query: str) -> str
  Searches past events using both search engines.
  - SQLite keyword search (LIKE queries)
  - ChromaDB semantic search (vector similarity)
  - Returns up to 5 results from each source

Tool 4: generate_summary() -> str
  Generates a session summary report.
  - Retrieves stats from SQLite (total frames, alerts, suspicious count)
  - Lists all alerts by severity
  - Lists all suspicious frames with details
  - Returns formatted text summary

6.3 prompts.py — System Prompts
--------------------------------
SECURITY_AGENT_SYSTEM_PROMPT:
  Defines the agent's role as an elite AI Security Analyst. Instructs it to:
  - LOG every event using tools
  - EVALUATE alert rules
  - IDENTIFY patterns (repeat vehicles, loitering, escalation)
  - RECOMMEND actions based on risk level

SUMMARY_PROMPT:
  Template for generating end-of-session security briefings.

7. OBJECT DETECTION — detector.py
================================================================================

Class: VideoDetector
  YOLOv8 + DeepSORT real-time object detection and tracking system.

  __init__(model_path="yolov8n.pt", confidence=0.4)
    - Loads YOLOv8 nano model (auto-downloads if not present)
    - Initializes DeepSORT tracker
    - Defines 17 security-relevant COCO classes:
      person, bicycle, car, motorcycle, bus, truck, backpack, umbrella,
      handbag, suitcase, bottle, cup, laptop, cell phone, book, clock, vase

  process_frame(frame: np.ndarray) -> dict
    Input: Raw OpenCV frame (BGR numpy array)
    Process:
      1. Run YOLOv8 inference
      2. Filter detections by confidence and relevant classes
      3. Update DeepSORT tracker for persistent IDs
      4. Draw bounding boxes with color coding
    Output:
      - annotated_frame: Frame with drawn bounding boxes
      - active_tracks: List of tracked objects with IDs
      - raw_detections: List of all detections with confidence scores
      - frame_count: Processing counter

  Color Coding:
    Person:    Orange (255, 165, 0)
    Backpack:  Red (0, 0, 255)
    Vehicles:  Cyan (255, 255, 0)
    Other:     Green (0, 255, 0)

8. ALERT RULES ENGINE — rules_engine.py
================================================================================

8 Configurable Alert Rules:

  Rule 1: after_hours_person (HIGH)
    Trigger: Person detected between 22:00-06:00
    
  Rule 2: perimeter_breach (CRITICAL)
    Trigger: Person near fence + breach keywords (climb, cut, test, break)
    
  Rule 3: unauthorized_access (CRITICAL)
    Trigger: No badge/ski mask near server room, warehouse, or loading dock
    
  Rule 4: unknown_vehicle_night (HIGH)
    Trigger: Unrecognized vehicle after business hours
    
  Rule 5: vehicle_reentry (MEDIUM)
    Trigger: Same vehicle seen more than once (possible surveillance)
    
  Rule 6: suspicious_behavior (HIGH)
    Trigger: Keywords: loitering, photographing, circling, casing, watching
    
  Rule 7: door_anomaly (MEDIUM)
    Trigger: Door open/ajar when it should be closed
    
  Rule 8: unauthorized_parking (LOW)
    Trigger: Vehicle in unauthorized parking zone

Class: AlertManager
  evaluate_rules(frame_analysis: dict, context: dict) -> list[dict]
    Evaluates all 8 rules against a single frame analysis.
    Returns list of triggered alert dictionaries.

  get_alert_summary() -> dict
    Returns total alerts, grouped by severity.

9.1 sqlite_store.py — SQLite Structured Storage
-------------------------------------------------
Class: SQLiteStore
  File: data/security_events.db

  Tables:
    frames:
      frame_id (PK), timestamp, location, objects_detected (JSON),
      event_type, risk_level, is_suspicious, raw_description,
      analysis_summary, agent_response, created_at

    alerts:
      alert_id (PK), frame_id (FK), timestamp, rule_name, severity,
      message, location, created_at

    vehicle_log:
      id (PK), vehicle_description, frame_id (FK), timestamp,
      location, entry_count

  Key Methods:
    insert_frame(...)          Insert a processed frame
    insert_alert(...)          Insert a triggered alert
    log_vehicle(...)           Log vehicle sighting, return entry count
    query_by_object(type)      Search frames by object type (LIKE)
    query_by_time_range(s, e)  Search by time window
    query_by_location(loc)     Search by location
    query_suspicious_frames()  Get all suspicious frames
    get_all_alerts()           Get all alerts ordered by time
    get_vehicle_count(desc)    Get sighting count for a vehicle
    get_session_stats()        Get summary statistics
    clear_all()                Reset all data for new session

9.2 chroma_store.py — ChromaDB Semantic Search
------------------------------------------------
Class: ChromaStore
  Directory: data/chroma_db/
  Collection: drone_frames

  Document Format:
    "Time: {timestamp}. Location: {location}. {description}
     Event type: {event_type}. Risk level: {risk_level}.
     Objects: {objects}."

  Key Methods:
    add_frame(...)                  Add frame to vector store
    semantic_search(query, top_k)   Natural language similarity search
    get_similar_frames(frame_id)    Find frames similar to a given frame
    get_frame_count()               Total indexed frames
    clear_all()                     Delete and recreate collection

10. WEB SERVER — server.py
================================================================================

Framework: FastAPI with Server-Sent Events (SSE)

API Endpoints:

  GET  /                        Serve main dashboard (index.html)
  GET  /api/status              Current simulation state (JSON)
  POST /api/reset               Force-reset simulation lock
  GET  /api/simulate            SSE — Start patrol simulation, stream results
  GET  /api/alerts              Get alerts, optional ?severity= filter
  GET  /api/frames              Get all processed frames
  GET  /api/frames/search       Search frames (?q=, ?risk=, ?time_filter=)
  GET  /api/semantic            ChromaDB semantic search (?q=)
  GET  /api/summary             Get stored summary
  POST /api/generate_summary    Generate fresh AI summary
  POST /api/analyze_video       SSE — Upload video for dual YOLO+VLM detection
  GET  /api/stats               Database statistics

Simulation Flow (GET /api/simulate):
  1. Resets state, initializes all components
  2. Creates SecurityAgent with LangGraph
  3. Loops through 30 frames:
     a. Generate telemetry
     b. Analyze frame with VLM (GPT-4o-mini)
     c. Process with LangChain agent (log, alert, query tools)
     d. Stream result via SSE
  4. Generate AI summary
  5. Stream completion event
  6. Always resets running flag (try/finally)

Video Analysis Flow (POST /api/analyze_video):
  1. Save uploaded video to temp file
  2. Extract up to 20 frames (2-sec intervals, 90% JPEG quality)
  3. For each frame:
     a. YOLOv8 detection (bounding boxes + confidence)
     b. GPT-4o-mini Vision analysis (contextual description)
     c. Merge YOLO objects + VLM objects (deduplicated)
     d. Run through security analysis pipeline
     e. Stream result via SSE
  4. Clean up temp file

12.1 main.py — Console Monitoring Pipeline
-------------------------------------------
Usage: python main.py

Flow:
  1. Initialize SQLite, ChromaDB, AlertManager, TelemetrySimulator
  2. Create SecurityAgent
  3. Process 30 frames sequentially (4-second delay between frames)
  4. For each frame: telemetry -> VLM analysis -> agent processing -> print

12.2 query.py — Event Query CLI
---------------------------------
Usage: python query.py [options]

Options:
  --object, -o "truck"       Search by object type (SQLite LIKE)
  --time, -t "after 22:00"   Search by time range
  --search, -s "suspicious"  Semantic search (ChromaDB)
  --alerts, -a               Show all triggered alerts
  --summary                  Show session statistics
  --suspicious               Show all suspicious events
  --location, -l "Main Gate" Search by location
